WHAT ARE UNIT TESTS?

  • Tests for fundamental pieces of your code
  • Important foundation of TDD
  • Less about finding bugs, more about preventing them
  • Typically automated using some framework

ADVANTAGES

TIME SAVINGS

  • A little time investment at beginning saves a lot of time in the long run (and even in the short run) (add: some facts)
  • Helps test every possible path and edge case  (testing these are normally very time consuming)

ADVANTAGES (cont'd)

QUALITY
  • Improves reliability of refactoring (think Agile methodology)
  • Encourages separation of concerns
  • Indicates design issue (if code can't be unit tested, it may benefit from a redesign)
  • Can disallow check-ins/commits when tests don't pass
  • "Push-Pull relationship" makes code sturdy (Add: explain this better or remove)

ADVANTAGES (cont'd)

DEVELOPER MORALE

  • Increases confidence
  • Helps you to think of your code as discreet parts
  • Assists code maintenance (e.g. write test for qa issue)
  • Helps developers communicate (tests are a kind of self-documentation)

EASE FUNCTIONAL CHANGES

  • Example: add date format
  • Update tests accordingly
  • Keep tests / implementation in sync
  • Makes codebase "sturdy"
  • Self documents

MOCKING / DEPENDENCY INJECTION

  • Ability to mock is important to testing individual units
  • Mocking lends itself to technique of DI
  • Example of mocking
  • DI also encourages separation of concerns
  • Should mock db layer, external resources, etc (data-independence) 
  • Good practice: no configuration in tests projects (improves maintainability and separation)
